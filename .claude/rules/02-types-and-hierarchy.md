# Типы страниц и иерархия

## Типы страниц ГДД

### Система
Группа связанных механик с общей целью. Пример: Боевая система, Прогрессия, Экономика.
- Содержит: цель, список дочерних механик, связи с другими системами, границы
- НЕ содержит: конкретные формулы, числа, edge cases

### Механика
Одно правило поведения внутри системы. Пример: Здоровье, Возрождение, Опыт.
- Содержит: назначение, логику, параметры (коды), формулы, состояния, edge cases, обратную связь
- Одна механика = одна страница. Не дублируй информацию — ссылайся.

### Фича
Элемент интерфейса или взаимодействие. Пример: HUD, экран входа, меню.
- Содержит: назначение, макет/вайрфрейм, элементы интерфейса, состояния экрана

### Контентная единица
Конкретная сущность в мире. Пример: корабль VALKYRION, NPC Echo, лазер LG-1.
- Содержит: сводную таблицу, параметры с кодами, варианты, поведение

## Правила размещения

- Система → первый уровень внутри «Системы»
- Механика → дочерняя страница системы
- Фича → внутри «Интерфейс»
- Контентная единица → внутри соответствующего каталога
- Новый NPC → дочерняя страница «Противники — Каталог»
- Новый корабль → дочерняя страница «Корабли — Каталог»
