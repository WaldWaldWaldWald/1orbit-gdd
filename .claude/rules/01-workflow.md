# Порядок работы с ГДД

## При создании новой страницы

1. Определи тип страницы: Система, Механика, Фича, Контентная единица
2. Прочитай `knowledge/mechanics-registry.md` — найди все связанные механики
3. Прочитай связанные страницы в Confluence (минимум страницу-родителя и все страницы, на которые будешь ссылаться)
4. Определи место в дереве ГДД (см. правило 02)
5. Сформируй название (русский, ≤6 слов, без глаголов)
6. Напиши текст, проверяя каждое утверждение на противоречия с существующими страницами
7. Перед отправкой — пройди чек-лист DoR (минимум R1-R9)

## При редактировании существующей страницы

1. Прочитай текущую версию страницы ПОЛНОСТЬЮ
2. Прочитай `knowledge/mechanics-registry.md` для этой механики — проверь инварианты
3. Вноси ТОЛЬКО запрошенные изменения. Не «улучшай» остальной текст
4. Покажи diff: что было → что стало → почему
5. Жди подтверждения от пользователя

## Запреты при работе

- НЕ выдумывай параметры, формулы или правила, которых нет в источниках
- НЕ добавляй «по аналогии с другими играми» — только то, что описано в Контексте
- НЕ меняй существующие формулировки без явного запроса
- НЕ используй слово «щит» (Shield, SH, ShieldAbs) для NPC
- НЕ вводи новые валюты, фракции или типы прогрессии
- НЕ описывай техническую реализацию (код, API, серверы, БД)

## При обнаружении противоречия

Если новый текст противоречит существующей механике:
1. СТОП — не пиши дальше
2. Процитируй оба утверждения (существующее и новое)
3. Спроси пользователя: какое верное?
4. Запиши решение в `knowledge/decisions-log.md`
