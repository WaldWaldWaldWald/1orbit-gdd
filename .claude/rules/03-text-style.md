# Правила оформления текста

## Стиль

- Русский язык. Латиница только для: имён сущностей (Echo, VALKYRION), аббревиатур (HP, DPS, NPC, HUD, UI, PvP, TTK, DoT)
- Максимум 20 слов в предложении
- Прямой порядок слов: субъект → действие → объект
- Настоящее время, третье лицо: «Пилот получает урон» (не «получит» и не «получал»)
- Без вводных слов: убрать «однако», «тем не менее», «при этом», «также», «кроме того»
- Без оценочных слов: убрать «быстро», «мощный», «значительный», «сильный»
- Без неопределённых количеств: убрать «несколько», «множество», «некоторые» — заменить конкретными числами или кодами параметров

## Запрещённые конструкции

| Плохо | Хорошо |
|-------|--------|
| «быстро восстанавливается» | «восстанавливается за RegenTime сек» |
| «наносит большой урон» | «наносит BaseDamage ед. урона» |
| «несколько типов NPC» | «12 типов NPC» |
| «игрок может» | «пилот может» (если в игре) |
| «система рассчитывает» | «формула определяет» |

## Структура секций

Каждая страница механики содержит секции в этом порядке:
1. Вводный абзац (1-3 предложения: что это и зачем)
2. Назначение (зачем игроку эта механика)
3. Как работает / Логика
4. Параметры (таблица)
5. Формулы (блоки кода)
6. Обратная связь (таблица: Событие → Визуал → Звук)
7. Edge Cases (таблица: Ситуация → Поведение, минимум 5 шт.)
8. Связанные документы (ссылки с пояснением типа связи)
9. Дополнительно (обоснование дизайна, референсы) — необязательно

## Ссылки

- На другую страницу ГДД: _курсив с точным названием страницы_
- На внешний источник: [название](URL)
- На параметр: \`КодПараметра\` (моноширинный)
- На таблицу баланса: «Балансовые данные: [Лист: X, параметры: Y, Z]»

## Изображения и диаграммы

- Для механик с >2 состояниями — обязательна таблица переходов: Из → Триггер → В → Действие
- Для фич — обязательна ссылка на макет/вайрфрейм
- Подпись под изображением обязательна
