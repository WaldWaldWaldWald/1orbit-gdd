# Известные ошибки Claude при работе с ГДД 1ORBIT

Этот файл — паттерны ошибок, которые Claude допускал. Перед генерацией текста — перечитай и убедись, что не повторяешь.

## Формат записи

```
### Ошибка: [краткое описание]
Что произошло: [описание]
Почему это ошибка: [какой инвариант нарушен]
Как избежать: [конкретное действие]
```

---

### Ошибка: Добавление щита NPC
Что произошло: При генерации текста механики Claude написал «NPC поглощает урон щитом» / «Shield NPC» / «SH у NPC».
Почему это ошибка: Инвариант «NPC без щита» — NPC имеют только HP. Щит (SH, ShieldAbs) у NPC НЕ используется.
Как избежать: Перед написанием текста о NPC — проверить инвариант в `04-protection.md`. Если в тексте появляется слово «щит» рядом с «NPC» — это красный флаг.

### Ошибка: «Улучшение» формулировок при редактировании
Что произошло: При просьбе добавить один параметр Claude переписал соседние абзацы, изменив смысл.
Почему это ошибка: Существующий утверждённый текст неприкосновенен. Claude должен менять ТОЛЬКО то, что попросили.
Как избежать: При редактировании — показать diff (было→стало) для КАЖДОГО изменения. Не трогать текст, который не относится к запросу.

### Ошибка: Выдумывание правил «по аналогии»
Что произошло: Claude добавил механику из другой игры (не из Контекста 1ORBIT), потому что «так обычно делают в MMO».
Почему это ошибка: ГДД 1ORBIT определяется Контекстом. «По аналогии» = выдумка.
Как избежать: Каждое утверждение в ГДД должно быть обосновано ссылкой на Контекст или решением из `decisions-log.md`. Если основания нет — спроси пользователя.
